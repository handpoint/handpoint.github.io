Use the Handpoint terminal for payments or only token generation

Various payment types

Handpoint supports various different payment types, transaction types and capture methods. Including both host capture without the need for a settlement file and a batch capture where a settlement file is sent at the end of the day from the Handpoint HiWay.

Handpoint thus never stores any transaction data on the terminals, even when using batch capture, as the Handpoint gateway (HiWay) handles the end of day batch submission.

A list of supported transaction and payment types is here below:

  • Generate a token only (no sale)
  • Sale with returned token
  • Sale
  • Reversal
  • Refund
  • Refund reversal
  • Tip adjustment
  • In-line tipping, tip added at end of sale
  • Pre-Authorisations*
